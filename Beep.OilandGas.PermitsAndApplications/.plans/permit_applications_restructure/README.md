# Permit and Applications Project Restructure Plan

## Overview
This plan details a phased approach to fully restructure and revise the Beep.OilandGas.PermitsAndApplications project, with a focus on eliminating duplicate PPDM39 tables, aligning with canonical models, and modernizing the project structure and processes. Each phase is documented in a separate file. A TODO tracker is included to monitor progress.

## Enhancement plan (oil & gas workflows)
For **regulatory workflow maturity** (multi-agency intake, RFI, conditions, LifeCycle integration, compliance reporting), start here:

- **[ENHANCEMENT_PLAN.md](./ENHANCEMENT_PLAN.md)** — goals, industry reference table, themes A–E, success criteria  
- **[MASTER-TODO-TRACKER.md](../../MASTER-TODO-TRACKER.md)** — single master checklist (restructure + enhancement)

## Phases
- Phase 1: Audit & Remove Duplicate PPDM39 Tables
- Phase 2: Refactor Code to Use Canonical Models
- Phase 3: Redesign Extension Tables & Domain Logic
- Phase 4: Service, Mapper, and API Refactor
- Phase 5: Testing, Validation, and Documentation

See individual phase files for detailed steps.
