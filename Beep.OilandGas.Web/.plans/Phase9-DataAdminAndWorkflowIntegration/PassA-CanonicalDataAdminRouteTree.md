# Phase 9 Pass A — Canonical Data/Admin Route Tree

## Objective

Choose one route tree and one ownership model for PPDM setup, schema, metadata, quality, audit, versioning, and admin experiences across the solution.

---

## Whole-Solution Workstreams

| Workstream | Project Groups | Output |
|------------|----------------|--------|
| Admin route ownership | Web, ApiService, Branchs, UserManagement | canonical route map and role-aware navigation plan |
| Data platform ownership | PPDM39, PPDM39.DataManagement, DataManager, PPDM.Models | setup/schema/metadata ownership map |
| Workflow/admin overlap cleanup | LifeCycle, business-process pages, compliance and accounting domains | clear separation between workflow pages and raw admin/data pages |
| Support result/admin audit | calculation and operational modules | identify result-history/admin surfaces that need canonical placement |

---

## Required Outputs

1. Canonical admin/data route tree.
2. Setup/schema/seeding ownership decision.
3. Role and navigation model for admin-only vs engineer-facing routes.
4. Duplicate page list with retirement or compatibility decisions.

---

## Exit Gate

No admin/data capability should have competing route trees or ambiguous ownership after this pass.
