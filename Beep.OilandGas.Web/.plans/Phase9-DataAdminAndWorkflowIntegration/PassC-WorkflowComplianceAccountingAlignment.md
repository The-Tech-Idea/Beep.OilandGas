# Phase 9 Pass C — Workflow, Compliance, and Accounting Alignment

## Objective

Align the support domains so approvals, permits, accounting, and process dashboards connect cleanly to the operational work introduced in Phase 8.

---

## Alignment Map

| Alignment Area | Participating Projects | Result |
|----------------|------------------------|--------|
| Permit-backed workflows | PermitsAndApplications, PDF rendering, DevelopmentPlanning, Decommissioning, Web, ApiService | permit tasks can be launched and tracked from domain workflows |
| Finance-backed workflows | ProductionAccounting, Accounting, LifeCycle, Web, ApiService | AFEs, allocations, reconciliation, and closeout flows tie back to domain actions |
| Workflow dashboards | LifeCycle, Branchs, Drawing, Web, ApiService | dashboards reflect real process state instead of isolated page logic |
| Data governance | PPDM39, PPDM39.DataManagement, DataManager, UserManagement | audit, validation, quality, and versioning align to governed admin routes |
| Result administration | calculation modules, production modules, admin/data pages | result history, exports, and supporting metadata live in the correct admin surfaces |

---

## Required Outputs

1. Workflow-to-permit linkage plan.
2. Workflow-to-accounting linkage plan.
3. Governance checklist for audit, validation, quality, and versioning.
4. Role-aware navigation and page-access alignment.

---

## Exit Gate

Support-domain pages should now reinforce the operational workflow model rather than creating a competing process model.
