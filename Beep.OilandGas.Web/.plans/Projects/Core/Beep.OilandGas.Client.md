# Beep.OilandGas.Client

## Snapshot

- Category: Core/client abstractions
- Scan depth: Medium
- Current role: shared client-side abstractions and integration support
- Maturity signal: support library rather than primary business domain

## Observed Structure

- Top-level folders: `App`, `Authentication`, `Connection`, `DependencyInjection`, `Exceptions`, `Models`
- The project includes implementation templates, developer guidance, and service-status notes, which suggests it supports broader client patterns rather than end-user workflows directly

## Representative Evidence

- Client dependency support: `DependencyInjection/`
- Auth and connection support: `Authentication/`, `Connection/`
- Guidance files: `CLIENT_IMPLEMENTATION_TEMPLATE.md`, `DEVELOPER_GUIDE.md`, `SERVICE_IMPLEMENTATION_STATUS.md`

## Planning Notes

- Phase 6 and 10 should keep this project in mind when trimming direct web references so shared client abstractions are not bypassed accidentally.
- If typed-client contracts are formalized further, this project is a candidate place to centralize shared client-side patterns.
- `Beep.OilandGas.Client/App/Services/Analysis/AnalysisService.GasLift.cs` should treat the live API as wrapper-based: `GetGasLiftPerformanceAsync` must use `/api/gaslift/performance/{wellUWI}`, `OptimizeInjectionAsync` now maps to `/api/gaslift/analyze-potential` with the repo's default `100..5000` Mscf/day analysis range and 50 points, and `DesignGasLiftAsync` now maps to `/api/gaslift/design-valves` with the repo's five-valve default plus a bounded casing-rating-based injection pressure; the old valve-spacing and troubleshoot calls remain a separate audit because they still lack a direct live owner.
- `Beep.OilandGas.Client/App/Services/Analysis/AnalysisService.Pipeline.cs` should treat the live API as `/api/pipelineanalysis/*`; `AnalyzePipelineAsync` now maps `PipelineAnalysisResult` back into the legacy `PIPELINE_FLOW_ANALYSIS_RESULT`, `CalculatePressureDropAsync` now maps `GAS_PIPELINE_FLOW_PROPERTIES` onto `CalculatePressureDropRequest` and folds `PressureDropResult` back into the legacy result contract, while the old capacity/integrity/design assumptions remain a separate compatibility audit.
- `Beep.OilandGas.Client/App/Services/Analysis/AnalysisService.WellTest.cs` now routes legacy build-up, drawdown, derivative, and interpretation calls through `/api/calculations/well-test` by wrapping `WELL_TEST_DATA` into `WellTestAnalysisCalculationRequest`; only the old raw-history method remains a separate audit because the live API surface exposes analysis history, not raw `WELL_TEST_DATA` history, one-for-one.
